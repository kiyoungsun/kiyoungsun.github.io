I. Introduction

1. Motivation/background

Existing 'metronome' apps usally make only 4 beats-based sounds. Also they show beats only with beeping sounds and glittering signals to users. However, If the timing of beats are represented in viewable and easier ways, then the metronome can make sense about the timing of beats and the rhythm for users. The purpose of this project is to make various metronome beats and rhythms, furthermore, the most ultimate purpose is to represent beats in easier and viewable ways with bouncing objects.

II. System descriptions

1. Signal flows

If you click 'play' buton, schedule() function implemented with its resolution option (if 'tempoBase' value is 0, then it is 4 beats based and schedule() function schedules notes divided in 16 notes and if 'tempoBase' value is 1, then it is3 beats based and shcedule() function schedule the notes divided in 12 notes). Then for each rhythm('noteResolution' variable), we distinguished all first beats with diffrent sound which has diffrent frequency value. And then represented the beats in balls to bound the walls to count the beat.

Also you can get the BPM value with tapping any keys on the keyboard. If you tap the key more than 5 times, than its BPM value would be selected as a tempo of the metronome. If you want to use metronome with your desired BPM, then you can click 'reset' button and get the BPM value from the slide bar.

2. Implementation details

Bouncing balls are located at the wall when the beat is beeping. And it run to the wall on the other end of the canvas, which shows that balls are bouncing on the wall with their own beats. Red balls show first beats of the rhythm and other black balls show rest beats of the rhythm. The number of balls are different with the type of the rhythm.
